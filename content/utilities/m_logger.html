Basics

It is used to log messages out of the application based on a given logging level. There are four logging levels available:

  • M.INFO
  • M.DEBUG
  • M.WARN
  • M.ERR
By defining a logging level, the developer can decide the importance of a logged message. The screenshot below shows the particular output on the JavaScript console (here in Chrome Developer Tools).

Methods
Name Parameters Return Type Description
log
  • msg (String): The logging message.
  • level (Number, optional): The logging level, as a number from 0 to 3. Can be passed via constant:
    • M.INFO: 0
    • M.DEBUG: 1
    • M.WARN: 2
    • M.ERR: 3
 Logs the passed message on the console. The type of output is defined via an optional passed logging level. If no level is passed, M.DEBUG is used by default. The method also takes care, that things don't blow up, when this object is used in browsers that don't support this or don't have a console.

Examples

The only thing you can do with a loader view is to show / hide it. So the following coe snippets show how to do this: 

/* logging an info message */
M.Logger.log("I'm an information", M.INFO);

/* logging a debug message */
M.Logger.log("I'm a debug message", M.DEBUG);

/* logging a warning message */
M.Logger.log("I'm a warning", M.WARN);

/* logging an error message */
M.Logger.log("I'm an error", M.ERR);

Basics

Handling events and being able to respond to a user's actions is essential to any UI based application. As a developer you need to be able to make UI elements listen to events, more precisely to DOM events, in order to perform a certain action once the corresponding event was fired.

Within the The-M-Project framework, we provide a component called M.EventDispatcher. This object is used to register DOM events for all UI components. This includes events that are typically fired by a certain user action, like tapping on a button (tap), and events that are fired by the framework itself, like indicating that a page will be shown now (pagewillshow). The available events depend on the particular view. A button, M.ButtonView, provides the tap event, a page provides the pageshow event.

How to bind events?

Binding events, no matter what type of events, is pretty easy and follows the same syntax all across the framework. M.View, which is the base object for all views, provides a propety called events. This is used to register a view's events out of an application. While the base structure of such an event registration is always the same, there are three different ways of assigning an action to a certain event:

  1. Specify a scope and a name of a method, that is available within this scope. M.ButtonView.design({ value: 'myButton', events: { tap: { target: myApp.myController, action: 'myMethod' } } });
  2. Specify a scope and a method (as an object), that will be executed within this scope. M.ButtonView.design({ value: 'myButton', events: { tap: { target: myApp.myController, action: function() { alert('Hello World'); } } });
  3. Specify a method (as an object), that will be executed within the window scope. M.ButtonView.design({ value: 'myButton', events: { tap: { action: function() { alert('Hello World'); } } });

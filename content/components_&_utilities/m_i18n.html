Basics

With M.I18N, The-M-Project provides a tool for internationalisation and localisation. It can be used to set and get the application's language and to localize any string within an application. Such a string, for example, can be a label, a url or a path to an image.

To get an idea of what is possible with this feature, take a look at our Todos demo application and switch the language.

Methods
Name Parameters Return Type Description
getLanguage String This method is used to get a language for the current user. This process is divided into four steps. If one step leads to a language, this one is returned. The steps are prioritized as follows:
  1. get a previously stored language for the user (set by setLanguage)
  2. get the user's language by checking his navigator
  3. use the application's default language
  4. use the systems's default language
isLanguageAvailable
  • language (String): The language to be checked.
Boolean This method checks if the passed language is available within the application or not. Normally you, as a developer, should know what language your application support. Anyway, you can use this method to learn whether a language is supported or not.
l
  • key (String): The key to the localized string.
  • context (Object): Optional. An object defining the values for a parameterized, dynamic translation.
String This method returns the localized string for the given key based on the current language. If context is passed, the input string is searched for pattern <%= the_key %> to be replaced by the property named the_key of context. With this, dynamic parts in translations can be achieved.
setLanguage
  • language (String): The language to be set.
This method sets the application's current language and forces it to reload. After the reload, the application will be displayed in the selected language.
Note: Even though we force a reload of the application, this features works offline, too! so don't hesitate to localize your app, you won't lose any functionality.

Example

Localizing a string

The most important feature of M.I18N is the localization of strings. Once you created your langauge files, you can simply call the l method and let The-M-Project do the magic. The following code snippets show you how to use this feature within an application. 

/* localizing a label */
label: M.LabelView.design({
    value: M.I18N.l('register')
})

/* now assuming sth. like this in a language file: */
/* 'greeting': 'Hello, <%= name %>!'  */

/* localizing a label parameterized */
label: M.LabelView.design({
    value: M.I18N.l('greeting', { name: 'Brendan' })
})
/* Result, e.g.: 'Hello, Brendan!' */


/* Result, e.g.: 'Register', 'Registrieren', 'Registro' */

/* localizing a button */
button: M.ButtonView.design({
    value: M.I18N.l('confirm')
})
/* Result, e.g.: 'Confirm', 'Bestätigen', 'Confirmar' */

/* localizing an image */
button: M.ButtonView.design({
    value: M.I18N.l('headerImage')
})
/* Result, e.g.: 'headerImage_en.png', 'headerImage_de.png', 'headerImage_es.png' */

Setting the application's language

Another common use case is that you want to set the application's language. Let's say you provide english, german and spanish translations for your application and want the user to be able to freely choose between these languages. Therefore you will need the setLanguage method to assign the language, chosen by the user, to the application. 

/* set the application's language to german */
M.I18N.setLanguage('de_de');

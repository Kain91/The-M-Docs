How to build a HelloWorld app

Get started with The-M-Project within a few minutes
Setup · Launch · Editing · Build

Setup a new project

First of all, make sure you downloaded Espresso&The-M-Project and did all the initial stuff described in the Get Started section on our website.

Now lets create a new project with Espresso. Therefore type the following into your command line ... 

espresso help

... to see all possible commands: 

Available commands:
build          Command to build a project
deploy         Command deploy projects
generate       Command to generate new or additional files for a project
init           Command line tool to generate a new project
package        Command to build native applications
server         Command to run the Development server
version        Command to print the Espresso version
help           Command to show help for a subcommand

The init command looks promising, don't you think? So lets see what it can do. 

espresso init help

We've got the following options: 

Available options:
-p, --project=PROJECT       Project name
-d, --directory=DIRECTORY   Specify a custom projects directory Default: $PWD
-e, --example               Generate an example "Hello World" application Default: false
-h, --help                  Show this help for command init Default: false

So we will need the -p option for specifying a project name and the -e option for setting up a hello world example. Let's do it. So first of all navigate to the directory where you want to init your project in, then run the just learned command: 

cd /path/to/folder
espresso init -e -p HelloWorldSample

Espresso now will initialize your project, copy all the required resources and set up some sample code, since we chose the -e option.

Launch the application

Ok, before we take a look at the source code, let's see how the app looks in the browser. As we've seen before, there is a command called server. Let's try this: 

cd /HelloWorldSample
espresso server

Now Espresso should start a development server (based on Node.js) and print the local URL: 

LOG: Server running at http://127.0.0.1:8000/HelloWorldSample

Open your web browser (Google Chrome or Safari are recommended) with the URL above and enjoy your first app. You can also test it with e.g. the iOS simulator or Android emulator. This is what it should look like:

Edit the code

So far, so good. Now let's edit the app. Open the project folder with an IDE of your choice. We recommend to use the amazing WebStorm IDE, but you're free to use whatever you want to.

For now, the only important thing inside your project folder is the app directory. This is where your application's source code is located. As you can see, there's a resources folder and a file called main.js. Inside that folder, you will find some CSS and images. Speaking for itself, this is where you will put the resources of you app. For now we will focus on the main.js: 

HelloWorldSample.app = M.Application.design({

    /* Define the entry/start page of your app. This property must be provided! */
    entryPage : 'page1',

    page1: M.PageView.design({

        childViews: 'header content footer',

        header: M.ToolbarView.design({
            value: 'HEADER',
            anchorLocation: M.TOP
        }),

        content: M.ScrollView.design({
            childViews: 'label',
            label: M.LabelView.design({
                value: 'Welcome to The M-Project'
            })
        }),

        footer: M.ToolbarView.design({
            value: 'FOOTER',
            anchorLocation: M.BOTTOM
        })

    })

});

This is the basic structure of an application consting of a header toolbar, content area (M.ScrollView) and a footer. Let's try to edit some values as follows: 

label: M.LabelView.design({
    value: 'What an awesome demo application'
})

Now switch to your browser and hit the refresh button. This will force Espresso to gather the app's source again and deliver the update version. So your app should look something like:

Build the app

It's time for the final step of this tutorial. Let's assume the app was deploy-ready now. Want you would want to do is to build the application to be able to deploy it on a web server. And guess who can help you with this? Espresso! 

espresso build

This will call Espresso to finally pack all the stuff together and save it in the build directory of your application. Every build gets its own build number, based on the current timestamp. So let's take a look at what just was created. 

cd build/

Inside of that directory you will find a folder containing the latest build. To publish this app, you now simply would have to put this folder on a web server and link it to some url. That's it.

Note: There is no special requirement to the web server. Especially Node.js is no requirement (since people sometimes get this wrong).

Basics

The M.ScrollView is a container for the scrollable content on a single page. It should be used as a wrapper for any content that isn't part of a header toolbar or footer toolbar/tabbar.

Note: It is very important to use this view for your scrollable content since this view adds some important meta information to this area so jQuery Mobile recognizes it as the page's content area.

Properties
Name Relevance Type Default value Description
childViews recommended String A string naming the child views of the scroll view. E.g. 
childViews: 'label button textfield'
Those childViews have to be defined inside the definition of the scroll view, as you can see within the samples.
cssClass optional String This property can be used to set CSS classes for the scroll view.

Example

Although the scroll view is a very important view for an application, it is pretty easy to use. 

M.ScrollView.design({
    childViews: 'label button',
    cssClass: 'myScrollView',
    label: M.LabelView.design({
        value: 'My Label'
    }),
    button: M.ButtonView.design({
        value: 'My Button'
    }),
});

CSS Styles

There actually only is one relevant CSS class when using a scroll view:

  • ui-content: This CSS class defines some basic styling (especially the padding) of an application's content area.

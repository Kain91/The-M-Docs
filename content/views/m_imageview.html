M.ImageView

A view presenting an image in your app.
Basics · Screenshots · Properties · Methods · Example · CSS Styles

Basics

The M.ImageView is a very basic view, that simply renders an image by using HTML's img element.

Properties
Name Relevance Type Default value Description
cssClass optional String This property can be used to set CSS classes for an image.
value mandatory (if computedValue is not used) String This property defines the value of the image, which means the source that is displayed. You can either provide a valid path or a data URI.

Events
Name Passed Properties Description
error
  • viewId (String): The Id of the image, e.g. 'm_502';
  • event (Object): The event object.
The event is fired when the image could not be loaded. This can happen if e.g. the source is not valid or there is no network connection available.
load
  • viewId (String): The Id of the image, e.g. 'm_502';
  • event (Object): The event object.
The event is fired when the image was successfully loaded.
tap
  • viewId (String): The Id of the image, e.g. 'm_502';
  • event (Object): The event object.
The event is fired when a user taps on the image.

Example

Display a 'local' image, located in e.g. base/images/. As you can see, all images will be available in theme/images/ once the app was built. So you can use the same path even if you are using device-specific images. 

M.ImageView.design({
    value: 'theme/images/myImage.png'
});

This example shows how to style an image using the cssClass property. 

M.ImageView.design({
    value: 'theme/images/myImage.png',
    cssClass: 'myImage'
});




/* CSS */
.myImage {
    width: 50%;
    border: 1px solid #FF0000;
}

M.ContainerView

A container for other views. Only renders a code frame no representation.
Basics · Screenshots · Properties · Methods · Example · CSS Styles

Basics

All child views get wrapped with a div element.

Properties
Name Relevance Type Default value Description
childViews optional String A string naming the child views of the container. E.g. 
childViews: 'teaser main footer'
Those childViews have to be defined inside the definition of the container, see the examples.
cssClass optional String This property can be used to set CSS classes for a container.

Methods

There are no public methods available.

Example

 

M.ContainerView.design({
        cssClass: 'greyBox'
})

M.ContainerView.design({
        cssClass: 'greyBox',
        childViews: 'teaser main footer',
        teaser: M.ContainerView.design({
                cssClass: 'teaser'
        }),
        main: M.ContainerView.design({
                cssClass: 'main'
        }),
        footer: M.ContainerView.design({
                cssClass: 'footer'
        })
})

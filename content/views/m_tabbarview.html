M.TabBarView

A convenient navigation bar as known from e.g. the iPhone.
Basics · Screenshots · Properties · Methods · Example · CSS Styles

Basics

A tab bar view is a special variant of a toolbar at the top or bottom of a page, that consists of up to five horizontally aligned tabs. A M.TabBarView can be used as the top navigation level for an application since it is always visible an indicates the currently active part of the application by highlighting the corresponding tab.

Properties
Name Relevance Type Default value Description
anchorLocation recommended String Defines the position of the TabBar. Possible values are:
  • M.BOTTOM => is a footer tab bar
  • M.TOP => is a header tab bar
name optional String tab_bar This property defines the tab bar's name. This is used internally to identify the tab bar inside the DOM. If your application only uses one tab bar (the typical way), there is no need to specify a custom name.

Example

For setting up a 3 item tab bar, add a new file to your project's view directory called e.g. 'tabs.js' and paste the following code. For further information about the tab bar items, see the M.TabBarItemView. 

TabBarSample.TabBar = M.TabBarView.design({
    childViews: 'tab1 tab2 tab3',
    anchorLocation: M.BOTTOM,
    tab1: M.TabBarItemView.design({
        value: 'Tab 1',
        page: 'page1'
    }),
    tab2: M.TabBarItemView.design({
        value: 'Tab 2',
        page: 'page2'
    }),
    tab3: M.TabBarItemView.design({
        value: 'Tab 3',
        page: 'page3'
    })
});

Once the tab bar is set up, it needs to be added to any page you want it to be visible. But note that you will have to 'require' the tabs file first. So the following lines show a sample page using the above created tab bar. 

m_require('app/views/tabs.js');

TabBarSample.FirstPageView = M.PageView.design({
    childViews: 'header content tabBar',
    header: M.ToolbarView.design({
        value: 'Page 1'
    }),
    content: M.ScrollView.design({

    }),
    tabBar: TabBarSample.TabBar
});

As you can see in the sample above, there are two things to do:

  1. Use the m_require command to make sure the tab bar is loaded by JavaScript before your page's source is interpreted.
  1. Refer to the tab bar object (TabBarSample.TabBar) for adding it as the page's tab bar.

All the tab activating/deactivating and switching the page/tab stuff will happen automatically, covered by the framework.

CSS Styles

The styling of the tab bar is done by jQuery Mobile. It is pretty much the same as with the M.ToolbarView. Depending on the anchorLocation property, some of the following css classes are responsible for the layout:

  • ui-header / ui-footer: This defines some basic layout and positioning setup for the tab bar.
  • ui-bar-a: This class defines the color and visual style of the tab bar.
  • ui-header-fixed / ui-footer-fixed: This indicates that the tab bar is fixed.

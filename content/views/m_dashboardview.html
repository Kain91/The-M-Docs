M.DashboardView

An icon-based startup screen for your application.
Basics · Screenshots · Properties · Methods · Events · Example · CSS Styles

Basics

A dashboard view displays images and a corresponding text in a grid-like view and serves as the homescreen of an application. By tapping on of the icons, a user can access certain features of an app. By default, there are three icons in a row and three rows per page possible. But you can easily adjust this to your custom needs.

Properties
Name Relevance Type Default value Description
childViews optional String This property can be used to statically assign dashboard items to the dashboard. For a more dynamic way use content binding.
contentBinding optional Array With this property a dynamic content (dashboard items) can be bound to the dashboard view. Everytime the bound property changes, the dashboard view will automatically update its items and re-render itself.
cssClass optional String This property can be used to set CSS classes for the dashboard.
isEditable optional Boolean NO This property can be used to specify whether or not the dashboard can be re-arranged by a user.
itemsPerLine optional Number 3 This property can be used to customize the number of items a dashboard shows per line. By default this is set to three.
name optional String dashboard This property defines the dashboard's name. This is used internally to identify the dashboard inside the DOM.
Note: If you are using more than one dashboard inside your application, make sure you provide different names.

Events

There are no actual events for the dashboard view itself. All events are based on the dashboard's items. But you can use the tap/click event of the dashboard view to handle them globally for all items instead of adding special behaviour to each dashboard item.

Note: You can also combine these global events with individual events for some/all dashboard items. Once a user clicks/taps an item, both events are fired (local first).

Name Passed Properties Description
click
  • domID (String): The DOM id of the tapped dashboard item.
This event is fired when a user clicks/selects a dashboard item.
tap
  • domID (String): The DOM id of the tapped dashboard item.
This event is fired when a user taps/selects a dashboard item. We recommend to use this event instead of click!

Example

Setup a dashboard view with content binding: 

M.DashboardView.design({
    contentBinding: {
        target: DashboardSample.ApplicationController,
        property: 'items'
    }
})

This is what the content binding could look like: 

var items = [{
    icon: 'theme/images/icon_settings.png',
    label: 'Settings',
    value: 'settings'
},
{
    icon: 'theme/images/icon_edit.png',
    label: 'Edit',
    value: 'edit'
},
{
    icon: 'theme/images/icon_wifi.png',
    label: 'WiFi',
    value: 'wifi'
}]

Setup a configured dashboard view: 

M.DashboardView.design({
    itemsPerLine: 3,
    isEditable: NO,
    contentBinding: {
        target: DashboardSample.ApplicationController,
        property: 'items'
    }
})

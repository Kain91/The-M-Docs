M.ToggleView

An easy way to toggle between two views
Basics · Screenshots · Properties · Methods · Example · CSS Styles

Basics

M.ToggleView defines the prototype of any toggle view. A toggle view accepts exactly two child views and provides an easy mechanism to toggle between these two views. An easy example would be to define two different button views that can be toggled, a more complex scenario would be to define two content views (M.ScrollView) with own child views and toggle between them.

Properties
Name Relevance Type Default value Description
type String M.ToggleView The type of this object.
isInFirstState Boolean YES States whether the toggle view currently displays its first child view or its second child view.
toggleOnClick optional Boolean NO Determines whether to toggle the view on click. This might be useful if the child views are e.g. buttons.
currentView M.View Contains a reference to the currently displayed view.

Methods
Name Parameters Return Type Description
toggleView This method toggles the child views by first emptying the toggle view's content and then rendering the next child view by calling renderUpdateChildViews().
setView
  • view (Object|String): The corresponding view.
This method can be used to set on of the toggle view's child views as the active one. Simply pass the view, its id or its name. If a view or id is passed, that does not match on of the toggle view's child views, nothing will be done.

Example

As described, there are two possibilities to use M.ToggleView.

  1. A simple example using the toggleOnClick property. 
    toggle: M.ToggleView.design({
    childViews: 'button1 button2',
    toggleOnClick: YES,
    button1: M.ButtonView.design({
        value: 'Button 1'
    }),
    button2: M.ButtonView.design({
        value: 'Button 2'
    })
})
  2. A more complex example. The toggle function is handled by a controller. 
    childViews: 'toggle button'
toggle: M.ToggleView.design({
	childViews: 'content1 content2',
	content1: M.ScrollView.design({
	    childViews: 'label textfield grid',
	    label: M.LabelView.design({
	        value: 'I am the label of the first view.'
	    }),
	    textfield: M.TextFieldView.design({
	        isGrouped: NO,
	        value: 'I am the textfield of the first view.'
	    }),
	    grid: M.GridView.design({
	        layout: M.TWO_COLUMNS,
	        childViews: 'button1 button2',
	        button1: M.ButtonView.design({
	            value: 'Button 1'
	        }),
	        button2: M.ButtonView.design({
	            value: 'Button 2'
	        })
	    })
	}),
	content2: M.ScrollView.design({
	    childViews: 'label',
	    label: M.LabelView.design({
	        value: 'Now it is all gone. I am all alone on the second view.'
	    })	
	})
}),

button: M.ButtonView.design({
    value: 'Toggle views',
    events: {
        tap:{
            target: exampleApp.Controller,
            action: 'toggleViews'
        }
    }
})

    The controller function could look like:

    toggleViews: function() {
	M.ViewManager.getView('pageView', 'toggle').toggleView();
}

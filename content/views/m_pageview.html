M.PageView

The highest-leveled view of an application, the root of any screen.
Basics · Screenshots · Properties · Methods · Events · Example · CSS Styles

Basics

The M.PageView is used in every application built with The-M-Project. It is the container view for every view on one screen. This means, every page defines a screen in your application. No methods of the PageView are available for public use. Beside the childViews property, there are also no properties to be used by the application developer. But there are several important events that are fired in the context of the page.

Example

This is as simple example of a page consisting of a header bar, a content area and a footer bar. Those three container views will then again contain views. 

    M.PageView.design({
        events: {
            pageshow: {
                target: MyApp.MyController,
                action: 'init'
            }
        }

        /* CSS class(es) assigned to the page */
        cssClass: 'page',

        childViews: 'header content footer',

        header: M.ToolbarView.design({
            ...
        }),

        content: M.ScrollView.design({
            ...
        }),

        footer: M.ToolbarView.design({
            ...
        })
    });

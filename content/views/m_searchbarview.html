M.SearchBarView

A special text field representing a search bar.
Basics · Screenshots · Properties · Methods · Events · Example · CSS Styles

Basics

M.SearchBarView defines a search bar that can be used inside of a list view or independently as a plain input field with a search styling. Basically a search bar view is the same as a default text field view except it is styled with a goggles icon at the left and there appears a 'delete' button at the right once you entered some text to reset the text field.

Methods
Name Parameters Return Type Description

Events
Name Passed Properties Description
blur
  • id (String): The search bar's id.
  • event (Object): The fired event.
This event is fired whenever the search bar view loses the focus.
enter
  • id (String): The search bar's id.
  • event (Object): The fired event.
This event is fired when a user hits the enter button while the search bar view has got the focus.
focus
  • id (String): The search bar's id.
  • event (Object): The fired event.
This event is fired whenever the search bar view gets the focus.
keyup
  • id (String): The search bar's id.
  • event (Object): The fired event.
This event is fired when a user hits any button while the search bar view has got the focus.
Note: By accessing the keyCode property of the passed event object, you can get the information which key was pressed.

Example

Setup a basic search bar view. 

M.SearchBarView.design({
})

Setup a basic search bar view with an initial text with special styling. 

M.SearchBarView.design({
    initialText: 'Keyword...',
    cssClassOnInit: 'initialText'
})

Get the search bar's value out of a controller. 

var value = M.ViewManager.getView('myPage', 'mySearchbar').value;

Use content binding reverse to automatically pass value updates to a controller property. Whenever the user hits a key on the keyboard and updated the value of the search bar view, the controller property is updated, too. So you could e.g. set the content binding of a label to this property to let it be auto updated. 

M.SearchBarView.design({
    contentBindingReverse: {
        target: MyApp.MyController,
        property: 'searchString'
    }
})

Listen to the keyup event to do something whenever the search bar's value is updated. 

M.SearchBarView.design({
    events: {
        keyup: {
            target: MyApp.MyController,
            action: 'updateSearch'
        }
    }
})

CSS Styles

There are a few CSS classes responsible for the look and feel of the search bar view:

  • ui-shadow-inset: This defines an inner shadow for the text field view.
  • ui-input-search: This class defines the basic size and dimension of the search bar view.
  • ui-icon-searchfield: This class is responsible for positioning the goggles/clear icon inside the search bar view.
  • ui-corner-all: This adds round corners to the search bar view.
  • ui-body-c: This class defines some styling stuff like border, text color, text-shadow or the background.
